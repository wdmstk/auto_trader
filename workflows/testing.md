# testing.md - テストワークフロー

このワークフローは、プロジェクトの各層におけるテスト活動（単体、統合、シミュレーション、ストレス、取引所）を計画、実行、報告するための標準手順を定義します。

## 1. 分析 (Analysis)

*   **テスト対象の特定**: テストが必要なコンポーネント、機能、またはシステム全体を特定する。
*   **テスト目標の設定**: テストによって達成すべき具体的な目標（例: バグ発見、パフォーマンス検証、堅牢性確認）を明確にする。
*   **リスク評価**: テスト対象の変更がもたらす潜在的なリスクと、そのリスクを軽減するために必要なテストレベルを評価する。
*   **AI指示**: 「テスト対象とテスト目標を特定し、関連するリスクを評価してください。」

## 2. 計画 (Planning)

*   **テスト戦略の策定**: どの種類のテスト（ユニット、統合、シミュレーション、ストレス、取引所）をどの範囲で実施するかを決定する。
*   **テストケース設計**: 具体的なテストケース、テストデータ、期待される結果を設計する。
*   **テスト環境準備**: テスト実行に必要な環境（テストネット、シミュレーション環境など）を準備する。
*   **自動化計画**: 可能であればテストを自動化し、CI/CDパイプラインへの統合を計画する。
*   **AI指示**: 「テスト戦略とテストケースを設計し、テスト環境を準備してください。テストの自動化を検討してください。」

## 3. 実装 (Implementation)

*   **テストコード記述**: 計画されたテストケースに基づき、テストコード（`tests/`）を記述する。
*   **テストデータ準備**: テスト実行に必要なテストデータを準備または生成する。
*   **AI指示**: 「計画されたテストケースに基づいてテストコードとテストデータを記述してください。」

## 4. 検証 (Validation) - (このワークフロー自体が検証のフェーズです)

## 5. ドキュメンテーション (Documentation)

*   **テストレポート**: テスト結果（パス/フェイル、カバレッジ、パフォーマンスメトリクス）を文書化する。必要に応じて `docs/implementation/` にレポートを作成する。
*   **テスト戦略の更新**: プロジェクトのテスト戦略（`.clinerules/04-testing.md`）が最新であることを確認する。
*   **AI指示**: 「テスト結果を文書化し、テスト戦略ドキュメントが最新であることを確認してください。」

## 6. 完了チェックリスト (Completion Checklist)

- [ ] テスト計画で定義されたすべてのテストが実行されたことを確認した。
- [ ] すべてのテストケースが期待通りにパスしたことを確認した（失敗した場合は適切な措置が取られたことを確認）。
- [ ] テストカバレッジが目標を満たしていることを確認した。
- [ ] CI/CDパイプラインにテストが統合され、自動実行されることを確認した。
- [ ] テスト結果が正確に文書化され、関係者に共有されたことを確認した。
- [ ] テストを通じて発見されたバグが適切に追跡され、修正されたことを確認した。
- [ ] テストが既存の `base_policy.md` および `PROJECT_RULES.md` に準拠していることを確認した。
