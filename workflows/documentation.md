# documentation.md - ドキュメンテーションワークフロー

このワークフローは、プロジェクトのドキュメントを更新、改善、または新規作成するための標準手順を定義します。

## 1. 分析 (Analysis)

*   **対象特定**: 更新または作成が必要なドキュメント（ADR, Spec, ARCHITECTURE.md, README.md, `clinerules` など）を特定する。
*   **目標設定**: ドキュメントの目的（例: 明瞭性向上、情報の追加、構造化）を明確にする。
*   **既存情報の収集**: 既存のコード、ADR、Spec、および関連するIssueから情報を収集する。
*   **AI指示**: 「更新または作成が必要なドキュメントと目的を特定し、関連する情報を収集してください。」

## 2. 計画 (Planning)

*   **構造設計**: ドキュメントの構成、アウトライン、含めるべきセクションを計画する。
*   **AI指示**: 「ドキュメントの構造とアウトラインを設計してください。」

## 3. 実装 (Implementation)

*   **ブランチ作成**: `documentation/<issue-id>-<description>` の命名規則で新しいGitブランチを作成する。
*   **ドキュメント作成/更新**: 計画に従ってドキュメントを作成または更新する。既存のドキュメントスタイルガイド（`.clinerules/06-documentation.md`）と一貫性を遵守する。
*   **AI指示**: 「ドキュメントスタイルガイドに従ってドキュメントを作成または更新してください。」

## 4. 検証 (Validation)

*   **レビュー**: ドキュメントの内容が正確で、明確で、簡潔であることをレビューする。
*   **整合性チェック**: ドキュメントがコードや他のドキュメントと矛盾していないことを確認する。
*   **AI指示**: 「ドキュメントの内容をレビューし、コードや他のドキュメントとの整合性を確認してください。」

## 6. 完了チェックリスト (Completion Checklist)

- [ ] ドキュメントの目的が完全に達成されたことを確認した。
- [ ] ドキュメントが正確で、明確で、簡潔であることを確認した。
- [ ] ドキュメントが既存のコードや他のドキュメントと矛盾していないことを確認した。
- [ ] 既存のドキュメントスタイルガイドに準拠していることを確認した。
- [ ] レビュープロセスが完了し、必要な承認が得られたことを確認した。
- [ ] `CHANGELOG.md` にドキュメント変更のエントリが追加されたことを確認した。
