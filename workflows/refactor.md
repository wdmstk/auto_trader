# refactor.md - リファクタリングワークフロー

このワークフローは、既存コードの機能変更を伴わない改善（可読性、保守性、パフォーマンス向上など）を行うための標準手順を定義します。

## 1. 分析 (Analysis)

*   **対象特定**: リファクタリングが必要なコード領域（例: 低凝集度、高結合度、重複コード、不必要な複雑さ）を特定する。
*   **目標設定**: リファクタリングの具体的な目標（例: 可読性向上、パフォーマンス改善、モジュール化）を明確にする。
*   **影響評価**: リファクタリングが既存機能に与える潜在的な影響を評価する。
*   **既存テスト確認**: リファクタリング対象コードに十分なテストカバレッジがあることを確認する。不足している場合は、リファクタリング前にテストを追加する。
*   **AI指示**: 「リファクタリングの対象と目標を特定し、既存機能への影響を評価してください。十分なテストカバレッジがあることを確認してください。」

## 2. 計画 (Planning)

*   **ADR作成/更新**: リファクタリングがアーキテクチャ上の重要な変更を伴う場合、`docs/adr/` に新しいADRを作成または更新する。
*   **Spec更新**: リファクタリングによって既存のSpecが影響を受ける場合、`docs/specs/` の関連するSpecを更新する。
*   **タスク分解**: `TASKS.md` に関連するリファクタリングタスクを追加し、Technical DebtまたはCurrent Sprintに配置する。優先度と複雑度を推定する。
*   **段階的変更の計画**: 大規模なリファクタリングは、小さく独立した変更セットに分割し、段階的に実施する計画を立てる。
*   **AI指示**: 「リファクタリングの計画としてADRとSpecを更新し、`TASKS.md`を更新してください。段階的な変更を計画してください。」

## 3. 実装 (Implementation)

*   **ブランチ作成**: `refactor/<issue-id>-<description>` の命名規則で新しいGitブランチを作成する。
*   **コード修正**: 計画に従ってコードをリファクタリングする。機能変更を伴わない純粋なリファクタリングに限定する。既存のコーディングルール（`.clinerules/02-coding.md`）と型安全性（`mypy`）を厳格に遵守する。
*   **テストの維持**: 既存のテストがリファクタリング後も引き続きパスすることを確認する。
*   **AI指示**: 「計画とコーディングルールに従ってコードをリファクタリングし、既存テストがパスすることを確認してください。」

## 4. 検証 (Validation)

*   **既存テスト実行**: すべての既存テスト（ユニット、統合、シミュレーション、ストレス）を実行し、リファクタリングが既存機能の動作に影響を与えていないことを確認する。
*   **パフォーマンスベンチマーク**: パフォーマンス改善が目標である場合、ベンチマークテストを実施し、目標が達成されていることを確認する。
*   **品質チェック**: `ruff check`, `mypy src`, `pytest` を実行し、コード品質とテストカバレッジを確認する。
*   **CI/CDパイプライン**: GitHub Actionsでの自動テストがパスすることを確認する。
*   **AI指示**: 「既存テスト、パフォーマンスベンチマーク（必要な場合）、コード品質ツール、CI/CDパイプラインでリファクタリングの品質を検証してください。」

## 5. ドキュメンテーション (Documentation)

*   **ARCHITECTURE.md更新**: リファクタリングがアーキテクチャ構造に影響を与える場合、`ARCHITECTURE.md` を更新する。
*   **AI指示**: 「リファクタリングがアーキテクチャに影響を与える場合、`ARCHITECTURE.md`を更新してください。」

## 6. 完了チェックリスト (Completion Checklist)

- [ ] リファクタリングの目標が達成されたことを確認した。
- [ ] 既存機能の動作が変更されていないことを確認した。
- [ ] 関連するADRおよびSpecが最新であることを確認した。
- [ ] `TASKS.md` に関連するタスクが完了済みとしてマークされていることを確認した。
- [ ] すべての自動テスト（ユニット、統合、シミュレーション、ストレス）がパスしていることを確認した。
- [ ] コード品質ツール（`ruff`, `mypy`）によるチェックがパスしていることを確認した。
- [ ] リファクタリングが既存の `base_policy.md` および `PROJECT_RULES.md` に準拠していることを確認した。
- [ ] レビュープロセスが完了し、必要な承認が得られたことを確認した。
- [ ] `CHANGELOG.md` にリファクタリングのエントリが追加されたことを確認した。
- [ ] 関連するドキュメント（`ARCHITECTURE.md`など）が更新されたことを確認した。
