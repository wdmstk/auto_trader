# feature.md - 機能開発ワークフロー

このワークフローは、新しい機能開発をプロジェクトに導入するための標準手順を定義します。

## 1. 分析 (Analysis)

*   **要件定義**: 新機能のビジネス要件と技術要件を明確にする。
*   **影響分析**: 既存システムへの影響、依存関係、潜在的なリスクを評価する。
*   **実現可能性調査**: 技術的な実現可能性と必要なリソースを特定する。
*   **既存ドキュメント確認**: 関連する `ARCHITECTURE.md`, `ADR`, `Specs`, `AGENT.md` を確認し、方針との整合性を検証する。
*   **AI指示**: 「新機能の要件、影響、実現可能性を分析し、既存のアーキテクチャやポリシーとの整合性を確認してください。」

## 2. 計画 (Planning)

*   **ADR作成/更新**: 重要な設計判断やアーキテクチャ上の決定があれば、`docs/adr/` に新しいADRを作成または更新する。
*   **Spec作成**: `docs/specs/` に新機能の詳細な実装仕様（Spec）を作成する。目的、入力、出力、前提条件、失敗モードと対策、テスト観点を明確に記述する。
*   **タスク分解**: `TASKS.md` に関連するタスクを追加し、Current SprintまたはBacklogに配置する。優先度と複雑度を推定する。
*   **ロードマップ更新**: `ROADMAP.md` に新機能が次のマイルストーンまたは将来の機能として反映されていることを確認する。
*   **AI指示**: 「新機能の設計計画としてADRとSpecを作成/更新し、`TASKS.md`と`ROADMAP.md`を更新してください。」

## 3. 実装 (Implementation)

*   **ブランチ作成**: `feature/<issue-id>-<feature-name>` の命名規則で新しいGitブランチを作成する。
*   **コード記述**: Specに従ってコードを実装する。既存のコーディングルール（`.clinerules/02-coding.md`）と型安全性（`mypy`）を厳格に遵守する。
*   **単体テスト**: 新しい機能ロジックに対して網羅的な単体テスト（`tests/`）を記述する。
*   **AI指示**: 「Specとコーディングルールに従ってコードを実装し、適切な単体テストを記述してください。」

## 4. 検証 (Validation)

*   **統合テスト**: 他のコンポーネントとの連携を確認する統合テストを実施する。
*   **シミュレーション/ストレステスト**: 必要に応じてシミュレーションテストやストレステストを実施し、堅牢性を確認する。
*   **品質チェック**: `ruff check`, `mypy src`, `pytest` を実行し、コード品質とテストカバレッジを確認する。
*   **CI/CDパイプライン**: GitHub Actionsでの自動テストがパスすることを確認する。
*   **AI指示**: 「統合、シミュレーション、ストレステストを実施し、コード品質ツールとCI/CDパイプラインで品質基準を満たしていることを検証してください。」

## 5. ドキュメンテーション (Documentation)

*   **README.md更新**: 必要に応じて `README.md` の「現在の実装ステータス」セクションを更新する。
*   **操作ガイド**: 新機能の運用に関するガイドやランブックが必要な場合、`docs/implementation/` に作成する。
*   **AI指示**: 「新機能に関連するドキュメント（README.md、操作ガイドなど）を更新してください。」

## 6. 完了チェックリスト (Completion Checklist)

- [ ] 新機能の要件が完全に満たされていることを確認した。
- [ ] 関連するADRおよびSpecが最新であることを確認した。
- [ ] `TASKS.md` に関連するタスクが完了済みとしてマークされていることを確認した。
- [ ] `ROADMAP.md` に新機能の進捗が反映されていることを確認した。
- [ ] すべての自動テスト（ユニット、統合、シミュレーション、ストレス）がパスしていることを確認した。
- [ ] コード品質ツール（`ruff`, `mypy`）によるチェックがパスしていることを確認した。
- [ ] 新機能が既存の `base_policy.md` および `PROJECT_RULES.md` に準拠していることを確認した。
- [ ] レビュープロセスが完了し、必要な承認が得られたことを確認した。
- [ ] `CHANGELOG.md` に新機能のエントリが追加されたことを確認した。
