# 07-git.md - Gitルール

## コミットメッセージ

*   **方針**: コミットメッセージは `type: subject (optional scope)` 形式に従い、簡潔かつ明確にする。
*   **AI指示**: 「Conventional Commits 仕様に従ってコミットメッセージを作成してください。変更の意図と内容を明確に伝えてください。」

## ブランチ戦略

*   **方針**: Git Flow または GitHub Flow に従い、フィーチャーブランチ、リリースブランチ、ホットフィックスブランチを適切に運用する。
*   **AI指示**: 「新しい機能やバグ修正には、`feature/<issue-id>-<feature-name>` または `bugfix/<issue-id>-<bug-description>` の命名規則でフィーチャーブランチを作成してください。メインブランチへの直接コミットは避けてください。」

## プルリクエスト（PR）

*   **方針**: 各PRは単一の論理的な変更セットを表し、明確な説明と関連するドキュメントへのリンクを含むべき。
*   **AI指示**: 「プルリクエストを作成する際は、変更内容のサマリ、目的、関連するIssueやADRへのリンク、テスト結果を明確に記述してください。コードレビューの促進を意識してください。」

## マージ戦略

*   **方針**: メインブランチへのマージは、`Squash and Merge` または `Rebase and Merge` を使用し、クリーンなGit履歴を維持する。Fast-forwardマージは避ける。
*   **AI指示**: 「フィーチャーブランチをメインブランチにマージする際は、`Squash and Merge` または `Rebase and Merge` を使用してコミット履歴を整理してください。マージコミットメッセージは、変更の全体像を要約するようにしてください。」
