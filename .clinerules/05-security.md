# 05-security.md - セキュリティルール

## 機密情報の管理

*   **方針**: APIキー、シークレット、その他の機密情報は、コードベースに直接ハードコーディングせず、環境変数やセキュアな設定管理システムを通じて管理する。
*   **AI指示**: 「APIキーや認証情報などの機密情報をコードに直接記述しないでください。代わりに環境変数またはセキュアな設定ファイル（例: `.env`、`config.prod.yaml`）から読み込むようにしてください。」

## 入力検証とサニタイズ

*   **方針**: ユーザー入力や外部からのデータは、常に検証とサニタイズを行い、クロスサイトスクリプティング（XSS）、SQLインジェクション、コマンドインジェクションなどの脆弱性を防ぐ。
*   **AI指示**: 「ユーザー入力や外部APIからのデータを受け取る際は、必ず厳格な検証とサニタイズを適用してください。予期せぬ文字やフォーマットの入力を拒否し、既知の脆弱性パターンを防ぐための処理を実装してください。」

## 最小権限の原則

*   **方針**: システムコンポーネント、サービス、およびAPIキーには、その機能を実行するために必要最小限の権限のみを付与する。
*   **AI指示**: 「各サービスやAPIキーに割り当てる権限は、その操作に必要最小限のものに限定してください。過剰な権限はセキュリティリスクを高めます。」

## 依存関係のセキュリティ

*   **方針**: プロジェクトの依存関係（ライブラリ、パッケージ）は定期的に監査し、既知の脆弱性がないことを確認する。
*   **AI指示**: 「使用しているライブラリやフレームワークに既知のセキュリティ脆弱性がないか、定期的に確認してください。最新のセキュリティパッチが適用されたバージョンを使用し、不要な依存関係は削除してください。」

## ログと監視

*   **方針**: セキュリティ関連イベント（認証失敗、不正アクセス試行、異常なシステム動作など）は、適切な詳細とともにログに記録し、リアルタイムで監視する。
*   **AI指示**: 「セキュリティに関する重要なイベント（ログイン試行、APIエラー、異常な取引活動など）は、構造化されたログに記録し、迅速に検知・対応できるように監視システムと連携させてください。」
